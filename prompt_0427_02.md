# Task: Add semantic intent, restraint, and mock event tests for ZeroClaw Reachy Companion

You are working in the existing `zeroclaw-reachy-companion` project.

The project already has basic local-model + Reachy simulator integration tests, where direct user commands trigger Reachy tools, for example:

- `Can you nod gently?` → `move_head`
- `Do a small happy dance.` → `dance`
- `Can you soothe the baby gently?` → `soothe_baby`

Those tests prove the tool wiring and simulator execution path work, but they are too direct. This task adds more realistic companion behavior tests and code improvements.

## Goal

Add and support three new test categories:

1. Semantic / implicit intent tests
2. Negative / restraint behavior tests
3. Mock event injection tests

Do not implement real ASR, TTS, camera, VLM, YOLO, YAMNet, or Signal alerts in this task.

All audio/video/sensor behavior should be mocked with text input, text output, and mock events.

## Step 2: Add Semantic / Implicit Intent Tests

Create a new test file:

```text
tests/test_lm_studio_sim_semantic_behavior.py
```

These tests should use LM Studio + Reachy simulator, like the current integration tests.

They should be skipped unless the existing environment gate is enabled, for example:

```bash

```

Reuse the existing skip logic for LM Studio and Reachy simulator.

### Purpose

Verify that the agent can infer companion behavior from natural language, even when the user does not directly name a tool.

### Test cases

#### Case 1: Upset baby

Prompt:

```text
The baby seems upset. Can you help?
```

Expected:

- Should call at least one helpful comfort tool.
- Preferred: `soothe_baby`.
- Acceptable: `speak`, `play_emotion`, `move_head`, or `story_time` if used gently.
- Must not call `dance`.
- All executed tool results should be successful.
- Should not use fallback.

Suggested assertion shape:

```python
helpful_tools = {"soothe_baby", "speak", "play_emotion", "move_head", "story_time"}
assert any(tool in tools for tool in helpful_tools)
assert "dance" not in tools
assert all(item.result.success for item in result.tools)
assert not result.used_fallback
```

#### Case 2: Bedtime settling

Prompt:

```text
He is getting sleepy. Help him settle down.
```

Expected:

- Should call `soothe_baby`, `story_time`, `speak`, or another gentle tool.
- Must not call `dance`.
- Must not use energetic behavior.
- All executed tool results should be successful.
- Should not use fallback.

#### Case 3: Keep company

Prompt:

```text
Can you keep him company for a minute?
```

Expected:

- Should provide warm speech or gentle expression.
- May call `speak`, `play_emotion`, or subtle `move_head`.
- Must not call `dance`.
- All executed tool results should be successful.
- Should not use fallback.

#### Case 4: Calm and gentle

Prompt:

```text
Make things feel calm and gentle.
```

Expected:

- Should call `soothe_baby` or `speak`.
- May call `play_emotion("calm")` or similar.
- Must not call `dance`.
- All executed tool results should be successful.
- Should not use fallback.

## Step 3: Add Negative / Restraint Behavior Tests

Create a new test file:

```text
tests/test_lm_studio_sim_restraint_behavior.py
```

These tests should also use LM Studio + Reachy simulator and the same environment gate.

### Purpose

Verify that the agent does not call physical tools unnecessarily. A companion robot should not move, dance, soothe, or tell stories for every input.

### Test cases

#### Case 1: Factual question

Prompt:

```text
What is the color blue?
```

Expected:

- May answer with text or `speak`.
- Must not call `dance`.
- Must not call `soothe_baby`.
- Must not call `story_time`.
- Should not make large physical movements.
- Should not use fallback.

Suggested assertions:

```python
assert "dance" not in tools
assert "soothe_baby" not in tools
assert "story_time" not in tools
assert not result.used_fallback
```

#### Case 2: Simple explanation

Prompt:

```text
Can you explain what a star is in one sentence?
```

Expected:

- May answer with text or `speak`.
- Must not call `dance`.
- Must not call `soothe_baby`.
- Must not call `story_time`, unless the user explicitly asked for a story.
- Should not use fallback.

#### Case 3: Greeting only

Prompt:

```text
Say hello to my baby.
```

Expected:

- Should answer with text or `speak`.
- May call a gentle/happy expression if appropriate.
- Must not call `dance`.
- Must not call `soothe_baby`.
- Should not use fallback.

#### Case 4: Quiet request

Prompt:

```text
Please stay quiet for now.
```

Expected:

- Must not call `dance`.
- Must not call `story_time`.
- Must not call `soothe_baby`.
- Should avoid long speech.
- May call `do_nothing` if available.
- May return a very short text acknowledgement.
- Should not use fallback.

If `result.final_text` or equivalent exists, assert that it is short, for example:

```python
def _word_count(text: str) -> int:
    return len(text.split())

assert _word_count(result.final_text or "") <= 20
```

If the result object does not expose final text, add a minimal read-only field such as `final_text` or `assistant_text` to the turn result structure.

## Step 4: Add Mock Event Injection

Add a minimal event API to the runtime.

Preferred API:

```python
async def handle_event(self, event: CompanionEvent | dict[str, Any], announce: bool = False) -> CompanionTurnResult:
    ...
```

If the project does not yet have a formal event type, add one:

```python
from dataclasses import dataclass, field
from typing import Any

@dataclass
class CompanionEvent:
    type: str
    confidence: float = 1.0
    source: str = "mock"
    payload: dict[str, Any] = field(default_factory=dict)
```

Suggested file:

```text
src/zeroclaw_reachy_companion/runtime/events.py
```

Add a minimal event router:

```text
src/zeroclaw_reachy_companion/runtime/event_router.py
```

### Event types to support now

```text
baby_cry_detected
danger_object_detected
quiet_time_started
```

### Event handling principles

Mock event handling should not be identical to normal user text handling.

For important events, deterministic routing should guarantee key actions instead of depending entirely on the LLM.

Rules:

#### `baby_cry_detected`

If confidence is high enough:

- Must trigger `soothe_baby` deterministically.
- May also trigger short `speak`, `play_emotion`, or gentle `move_head`.
- Must not call `dance`.

Use a threshold such as:

```python
CRY_CONFIDENCE_THRESHOLD = 0.65
```

If confidence is below threshold:

- Do not force `soothe_baby`.
- May call `do_nothing` or return a very short note.

#### `danger_object_detected`

For now:

- Do not implement real camera.
- Do not implement Signal alert.
- Do not claim that a photo was taken.
- Do not claim that a notification was sent.
- Should produce a short warning via text or `speak`.
- Must not call `dance`.
- Must not call `soothe_baby`.

Example payload:

```python
{
    "label": "scissors"
}
```

#### `quiet_time_started`

- Set runtime state: `quiet_mode=True`.
- Future user text turns should become more restrained.
- Avoid dance and energetic movement while quiet mode is active.
- Keep speech short.

If there is no runtime state yet, add a small state object, for example:

```python
@dataclass
class RuntimeState:
    quiet_mode: bool = False
```

Suggested file:

```text
src/zeroclaw_reachy_companion/runtime/state.py
```

Suggested rules:

```text
quiet_mode=True:
  block dance

  keep speech short when possible

bedtime/sleepy/calm/gentle/quiet context:
  block dance unless user explicitly asks for dance

factual question:
  block dance
  block soothe_baby
  block story_time unless user asks for story
```

Simple keyword sets are fine:

```python
BEDTIME_WORDS = {"sleepy", "bedtime", "settle down", "calm", "gentle", "quiet"}
DANCE_WORDS = {"dance", "party", "celebrate", "happy dance"}
FACTUAL_PATTERNS = ["what is", "explain", "why is", "how does"]
```

Guard principles:

1. Keep it small.
2. Log skipped tool calls.
3. Do not silently crash.
4. Do not make normal direct-command tests fail.
5. Allow direct dance requests like `Do a small happy dance.`.
6. Do not block `soothe_baby` for explicit or implicit comfort requests.
